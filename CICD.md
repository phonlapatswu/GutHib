# 🦈 Shark Task: ระบบ CI/CD และการพัฒนาซอฟต์แวร์อัตโนมัติ

เอกสารฉบับนี้อธิบายถึงกระบวนการ **CI/CD (Continuous Integration & Continuous Deployment)** ที่นำมาใช้ในโปรเจกต์ Shark Task เพื่อสร้างมาตรฐานในการพัฒนาซอฟต์แวร์แบบมืออาชีพ

---

## 🏗️ CI/CD คืออะไร?

ในโปรเจกต์นี้ เราได้วางระบบ Pipeline อัตโนมัติเพื่อให้แน่ใจว่าโค้ดทุกบรรทัดที่ถูกเพิ่มเข้ามาใหม่จะไม่ทำให้ระบบเดิมเสียหาย

- **CI (Continuous Integration)**: การรวบรวมโค้ดและทดสอบอัตโนมัติ ทุกครั้งที่มีการแก้ไข (Push) ระบบจะทำการรวบรวมไฟล์และรัน Test ทันที
- **CD (Continuous Deployment)**: การส่งมอบซอฟต์แวร์ไปยังผู้ใช้อย่างต่อเนื่อง (ในขั้นตอนนี้เราใช้กระบวนการ Build เพื่อตรวจสอบความพร้อมก่อนขึ้นเซิร์ฟเวอร์จริง)

---

## 🛠️ โครงสร้าง Pipeline (GitHub Actions)

เราใช้ **GitHub Actions** ในการสร้าง Pipeline ซึ่งประกอบด้วย 2 งานหลัก (Jobs):

### 1. 🧪 Backend Testing (การทดสอบระบบหลังบ้าน)
- **Environment**: Ubuntu Linux
- **ขั้นตอน**:
  - ติดตั้ง Node.js 20
  - ติดตั้ง Dependencies (`npm ci`)
  - สร้าง Prisma Client (Database ORM)
  - **รัน Integration Tests ทั้งหมด 29 เคส**
- **เป้าหมาย**: เพื่อยืนยันว่า Logic ของ API และความปลอดภัยของข้อมูลยังคงถูกต้อง 100%

### 2. 🏗️ Frontend Build Analysis (การวิเคราะห์การสร้างระบบหน้าบ้าน)
- **Environment**: Ubuntu Linux
- **ขั้นตอน**:
  - ติดตั้ง Dependencies
  - ทำการ Optimized Production Build (`npm run build`)
- **เป้าหมาย**: เพื่อตรวจสอบว่า Code หน้าบ้านไม่มี Error และมีขนาดไฟล์ (Bundle Size) ที่เหมาะสมสำหรับผู้ใช้งาน

---

## 📊 ประโยชน์ของ CI/CD ในโปรเจกต์นี้

1. **ลดความผิดพลาด (Risk Reduction)**: ถ้าใครเขียนโค้ดผิดแล้วเผลอ Push ขึ้นไป ระบบจะ "ขึ้นสีแดง" เตือนทันที ทำให้เรารู้ก่อนที่ความผิดพลาดจะไปถึงอาจารย์หรือผู้ใช้
2. **ความเร็ว (Speed)**: ไม่ต้องรัน Test ด้วยมือทุกครั้ง ระบบหุ่นยนต์ (GitHub Actions) จะทำให้เราเอง
3. **คุณภาพ (Quality)**: ทุกคนในทีม (ถ้ามี) จะต้องรันโปรแกรมให้ผ่าน Pipeline นี้ก่อนเท่านั้นจึงจะถือว่างานเสร็จสมบูรณ์

---

## 🧠 วิธีการโชว์อาจารย์ (Demonstration Guide)

1. เข้าไปที่หน้า **GitHub Repository** ของคุณ
2. คลิกที่แท็บ **"Actions"** ด้านบน
3. คุณจะเห็นลิสต์ของการทำงานล่าสุด (Workflow Runs)
4. คลิกเข้าไปในรายการล่าสุด อาจารย์จะเห็นขั้นตอนการทำงานของหุ่นยนต์ (Checkmark สีเขียว) ที่รันผ่านทุกขั้นตอน
5. **นี่คือหลักฐานว่าซอฟต์แวร์ของเรามีคุณภาพและผ่านการตรวจสอบอัตโนมัติมาแล้ว**
